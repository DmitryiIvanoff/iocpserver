В проекте представлен проксирующий TCP-сервер на основе IOCP с возможностью логирования.
Класс CMySQLRoutine - подпрограмма-синглтон, отвечает за работу с БД MySQL, при создании объекта создает n-потоков в которых
происходит обработка ивентов порта завершения. В редких случаях, если БД пытается передать большие данные, пишет напрямую в сокет клиента минуя CClientRoutine

Класс CClientRoutine - подпрограмма-синглтон, отвечает за работу с клиентом, также как и CMySQLRoutine создает n-потоков, в которых происходит обработка ивентов 
порта заверщения. Также в этом классе реализована очередь для предотвращения перемещивания данных и отправки данных не тому клиенту.

Оба класса "общаются" между собой посредством портов завершения.

В проекте присутствует минимальная обработка ошибок - в случае ошибки будет записан соответствующий erro лог, бефер удаляется из очереди, удаляем сессию - серевер продолжит обслуживать других клиентов.

Для тестов использовались PHP 7.0 , Apache 2.4 и MySQL 5.7
Скрипты для теста находятся в папке scripts: скрипт index.php - постит 10к запросов к БД в цикле и выводит результаты, скрипт stress_test.php также постит много запросов к БД - он 
дополнительно тестит прием больших данных от БД. Порт проксирующего сервера задается в скриптах, порт MySQL - в iocpserver.cpp.
Проект собирается по x86 архитектуру, на x64 не тестилось.
